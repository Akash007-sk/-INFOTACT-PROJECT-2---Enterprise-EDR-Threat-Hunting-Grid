# INTERNSHIP PROJECT REPORT

## Enterprise EDR & Threat Hunting Grid using Wazuh and Sysmon

---

# 1. Introduction

In today's digital world, cybersecurity has become an essential part of every organization. With the increasing use of internet-based services, cloud computing, and connected systems, the number of cyber threats has also increased significantly. Organizations face different types of cyber attacks such as malware infections, ransomware attacks, phishing, brute-force login attempts, and unauthorized system access. These attacks can cause severe damage to organizational data, reputation, and financial stability.

To protect systems and networks from these threats, organizations deploy Security Operations Centers (SOC). A SOC is responsible for monitoring, detecting, analyzing, and responding to security incidents in real time. Security monitoring systems collect logs from multiple machines and analyze them to identify suspicious behavior.

This internship project focuses on implementing an **Enterprise Endpoint Detection and Response (EDR) and Threat Hunting Grid** using modern cybersecurity tools. The system is designed to collect security logs from endpoint machines, analyze them for suspicious activities, and generate alerts when potential threats are detected.

The project uses **Wazuh** as the central security monitoring platform. Wazuh collects logs from multiple systems, analyzes them using predefined rules, and generates alerts for suspicious activities.

For advanced monitoring of Windows systems, **Sysmon** is integrated into the system. Sysmon provides detailed logging of system activities such as process execution, network connections, and file changes. These logs help detect malicious activities that may indicate cyber attacks.

The project also incorporates the **MITRE ATT&CK** framework, which helps security analysts understand attacker behavior by mapping alerts to specific tactics and techniques used by adversaries.

The goal of this project is to simulate a small enterprise security monitoring environment that demonstrates how modern SOC teams detect and respond to cyber threats.

---

# 2. Objectives of the Project

The primary objective of this internship project is to design and implement a centralized security monitoring system capable of detecting suspicious activities on endpoint machines.

Another important objective is to develop an Endpoint Detection and Response environment that continuously monitors system activities and generates alerts when potential threats are detected.

The project also aims to demonstrate the implementation of file integrity monitoring to detect unauthorized changes to critical system files.

Additionally, the project focuses on integrating system monitoring tools with the security platform in order to collect detailed system logs for analysis.

Another objective of the project is to simulate cyber attacks in a controlled environment and observe how the monitoring system detects and responds to these attacks.

Finally, the project aims to demonstrate how security alerts can be mapped to the MITRE ATT&CK framework to better understand the tactics and techniques used by attackers.

---

# 3. Scope of the Project

The scope of this project includes the design and implementation of a small-scale enterprise security monitoring system capable of detecting suspicious activities occurring on endpoint machines.

The system monitors logs generated by operating systems and security monitoring tools. These logs are analyzed by the central monitoring platform to detect potential security threats.

The project focuses primarily on endpoint security monitoring rather than full network intrusion detection. It demonstrates how system logs can be used to identify malicious activities such as unauthorized file modifications, suspicious process execution, and repeated login failures.

The project environment is created using virtual machines to simulate a small enterprise network environment. This allows the monitoring system to collect logs from multiple machines and analyze them centrally.

Although the system is implemented in a laboratory environment, the architecture and technologies used are similar to those used in real enterprise security operations centers.

---

# 4. Tools and Technologies Used

Several tools and technologies were used during the implementation of this internship project.

The primary platform used in this project is **Wazuh**. It is responsible for collecting logs from different systems, analyzing security events, and generating alerts when suspicious activity is detected.

The project also uses **Sysmon**, which is a powerful system monitoring tool that records detailed information about system activities. Sysmon captures events such as process creation, network connections, and file modifications.

For attack simulation and penetration testing, **Kali Linux** was used. Kali Linux provides various tools that can simulate cyber attacks in a controlled environment.

The security events detected in the system are categorized using the **MITRE ATT&CK**, which provides a standardized framework for understanding attacker techniques.

Virtualization software such as VMware or VirtualBox was used to create the testing environment where multiple machines could interact with each other.

---

# 5. System Architecture

The architecture of the project consists of three major components: the monitoring server, endpoint systems, and the attack simulation environment.

The monitoring server hosts the Wazuh manager and dashboard. It acts as the central system responsible for collecting and analyzing security logs from endpoint machines.

Endpoint systems are the machines that are being monitored. These systems have the Wazuh agent installed, which collects logs and sends them to the monitoring server.

Windows endpoint machines are also equipped with Sysmon to generate detailed system activity logs. These logs help identify suspicious behaviors that may indicate malicious activity.

The attack simulation environment represents an attacker system that attempts to compromise the monitored machines. This system generates suspicious activity that the monitoring system can detect.

All these components communicate with each other to form a centralized security monitoring environment similar to those used in enterprise organizations.

---

# 6. Deployment of the Monitoring Server

The monitoring server is the most important component of the system. It hosts the Wazuh manager, which receives logs from all connected endpoint machines.

During the project implementation, the monitoring server was deployed on a Linux-based system. After installation, the Wazuh manager began listening for incoming log data from endpoint agents.

The server also hosts the Wazuh dashboard, which provides a graphical interface for monitoring security events. Security analysts can use the dashboard to view alerts, analyze logs, and investigate potential security incidents.

Once the monitoring server was successfully configured, endpoint agents were connected to it so that system logs could be collected and analyzed.

---

# 7. Endpoint Agent Deployment

Endpoint agents were installed on the machines that needed to be monitored. These agents act as log collectors that gather information from the operating system and send it to the central monitoring server.

In this project, a Windows system was used as the primary endpoint machine. After installing the Wazuh agent, it was configured to communicate with the monitoring server.

Once the agent was successfully connected, the endpoint system started sending logs to the monitoring server. These logs included system events, authentication logs, and security-related activities.

The monitoring dashboard displayed the endpoint system as an active agent, confirming that the communication between the endpoint and the monitoring server was functioning correctly.

---

# 8. System Monitoring with Sysmon

Sysmon was installed on the Windows endpoint system to provide detailed monitoring of system activities.

Sysmon is capable of recording various types of system events, including process creation, network connections, file creation, and registry changes. These events provide valuable information for detecting malicious activities.

The logs generated by Sysmon were forwarded to the Wazuh agent so that they could be analyzed by the monitoring server.

By analyzing Sysmon logs, the monitoring system was able to detect suspicious behaviors such as unusual process execution or connections to suspicious network ports.

---

# 9. File Integrity Monitoring

File Integrity Monitoring was implemented to detect unauthorized changes to important system files.

This feature continuously monitors selected directories and files for any modifications. If a file is altered, deleted, or replaced, the system generates an alert.

File integrity monitoring is important because attackers often attempt to modify system files after gaining access to a machine. Detecting these modifications can help security teams identify potential compromises.

During the project implementation, critical system directories were configured to be monitored so that any changes would be immediately detected and reported.

---

# 10. Detection and Alerting Mechanism

The monitoring system analyzes incoming logs using a rule-based detection engine. Each rule is designed to identify specific types of suspicious activity.

For example, repeated login failures may indicate a brute-force attack attempt, while unusual process execution may indicate malware activity.

When a rule is triggered, the system generates a security alert. These alerts appear in the monitoring dashboard, allowing security analysts to investigate the incident.

Alerts are categorized based on severity levels so that critical threats can be prioritized and addressed quickly.

---

# 11. Active Response Mechanism

In addition to detecting threats, the system is capable of responding to certain security incidents automatically.

Active response mechanisms allow the monitoring system to take immediate action when suspicious activity is detected. For example, if an attacker repeatedly attempts to log in to a system, the monitoring platform can automatically block the attacker's IP address.

This automated response capability helps prevent attackers from continuing their activities and reduces the time required to mitigate threats.

By combining detection with automated response, the system becomes more effective in protecting endpoint systems.

---

# 12. Attack Simulation

To test the effectiveness of the monitoring system, attack simulations were conducted in a controlled environment.

An attacker machine running Kali Linux was used to simulate malicious activities such as repeated login attempts. These actions generated suspicious events that were captured by the monitoring system.

The monitoring platform successfully detected these events and generated alerts in the dashboard. This confirmed that the detection rules were functioning correctly.

Attack simulation is an important step in validating the effectiveness of security monitoring systems.

---

# 13. Results and Observations

The implemented monitoring system successfully detected multiple types of suspicious activities.

The system was able to detect repeated login failures, which could indicate brute-force attacks. It also detected unauthorized file modifications and suspicious system processes.

All detected events were displayed in the monitoring dashboard in real time. Security analysts could view detailed information about each event and investigate potential threats.

The integration with the MITRE ATT&CK framework provided additional insight into attacker techniques and behaviors.

Overall, the project demonstrated how centralized security monitoring systems can effectively detect and analyze cyber threats.
